# Cloud Computing

This document explains Cloud Computing and how it evolved from virtualization to modern cloud platforms like Google Cloud Platform (GCP).

---

## 📌 Big Picture Flow

Physical Servers → Virtualization → Virtual Machines → Cloud Computing → GCP

---

## 1. What is Cloud Computing?

### 🔹 Simple Definition
Cloud computing is the delivery of computing services like servers, storage, databases, and networking over the internet.

### 🔹 Professional Definition
Cloud computing is a model that enables on-demand access to shared computing resources over a network, which can be rapidly provisioned and released with minimal management effort.

---

## 2. Why Cloud Computing Was Needed

Even after virtualization, organizations still faced challenges like:

- Managing infrastructure manually
- Scaling systems quickly during demand spikes
- High upfront hardware cost
- Limited global accessibility

Cloud computing solved these problems by:
- Providing resources on demand
- Removing the need to own physical infrastructure
- Enabling global scalability

---

## 3. How Cloud Computing Works

Cloud providers own large data centers containing thousands of physical servers.

These servers run virtualization technology to create and manage virtual machines.

### 🔹 Flow
```
Cloud Provider Data Center
↓
Virtualization Layer (Hypervisors)
↓
Virtual Machines / Services
↓
Users access via Internet
```

---

## 4. Key Characteristics of Cloud Computing

- **On-Demand Access** → Resources available anytime
- **Scalability** → Increase or decrease resources quickly
- **Pay-as-you-go** → Pay only for what you use
- **Global Access** → Accessible from anywhere
- **Managed Infrastructure** → Provider handles hardware

---

## 5. Cloud Service Models

### 🔹 Infrastructure as a Service (IaaS)
> Provides virtual machines, storage, and networks

Example: Google Compute Engine

---

### 🔹 Platform as a Service (PaaS)
> Provides platform to deploy applications without managing infrastructure

Example: Google App Engine

---

### 🔹 Software as a Service (SaaS)
> Fully built applications delivered over the internet

Example: Gmail, Google Drive

---

## 6. Real-World Analogy

Think of cloud computing like electricity:

- You don’t build your own power plant
- You simply consume electricity when needed
- You pay based on usage

Similarly:
- You don’t build servers
- You use cloud resources on demand
- You pay only for what you use

---

## 7. Cloud Computing in Google Cloud (GCP)

Google Cloud Platform provides:

- Compute Engine → Virtual Machines
- Cloud Storage → Object storage
- BigQuery → Data analytics
- Cloud Run → Serverless containers

Google manages:
- Physical servers
- Networking
- Data centers
- Hypervisors

You manage:
- Applications
- Data
- Configurations

---

## 8. Advantages of Cloud Computing

- No hardware maintenance
- Fast scalability
- Lower upfront cost
- Global availability
- High reliability and uptime

---

## 9. Limitations of Cloud Computing

- Requires internet connection
- Potential vendor dependency
- Cost management complexity if not monitored

---

## 🔁 Summary

- Cloud computing delivers computing services over the internet
- It is built on top of virtualization
- It allows on-demand, scalable, and flexible resource usage
- It is the foundation of modern platforms like Google Cloud

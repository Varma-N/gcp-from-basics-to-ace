# Virtual Machines (VMs)

This document explains Virtual Machines, which are the result of virtualization and a core building block of cloud computing.

---

## 📌 Big Picture Flow

Physical Server → Hypervisor → Virtual Machines (VMs) → Cloud Infrastructure (GCP Compute Engine)

---

## 1. What is a Virtual Machine?

### 🔹 Simple Definition
A Virtual Machine (VM) is a software-based computer that behaves like a physical computer.

### 🔹 Professional Definition
A Virtual Machine is an isolated computing environment created by a hypervisor that emulates physical hardware, allowing an operating system and applications to run independently.

---

## 2. Why Virtual Machines Exist

Before VMs:
- One application per physical server
- Poor resource utilization
- Difficult scaling

With VMs:
- Multiple independent systems can run on one physical machine
- Better utilization of CPU, memory, and storage
- Easier scaling and management

---

## 3. How a Virtual Machine Works

A VM is created by dividing physical resources through a hypervisor.

### 🔹 Flow
```
Physical Hardware
↓
Hypervisor
↓
VM 1 VM 2 VM 3
(OS + Apps each running independently)
```

Each VM has:
- Its own operating system
- Its own applications
- Virtual CPU, memory, and storage

---

## 4. Key Characteristics of VMs

- **Isolation** → Each VM is independent
- **Portability** → Can be moved across systems
- **Scalability** → Can be created or deleted quickly
- **Flexibility** → Can run different OS on same hardware

---

## 5. Components of a Virtual Machine

- Guest Operating System (Linux / Windows)
- Virtual CPU (vCPU)
- Virtual Memory (RAM)
- Virtual Disk (Storage)
- Virtual Network Interface

---

## 6. Real-World Analogy

Think of a physical server as a building.

- Each Virtual Machine = an apartment
- Each apartment has its own electricity, water, and setup
- But all apartments share the same building infrastructure

---

## 7. Virtual Machines in Google Cloud (GCP)

In Google Cloud Platform:
- Virtual Machines are called **Compute Engine Instances**
- You can create VMs on-demand
- You only pay for what you use
- Google manages the underlying hardware and hypervisor

Example:
- Create a VM for a web server
- Create another VM for a database
- Both run independently in the cloud

---

## 8. Advantages of Virtual Machines

- Efficient use of hardware
- Easy scaling
- Cost-effective compared to physical servers
- Supports multiple operating systems
- Foundation of cloud computing

---

## 9. Limitations of Virtual Machines

- Slight performance overhead due to virtualization layer
- Requires hypervisor management
- Can consume significant resources if not optimized

---

## 🔁 Summary

- A Virtual Machine is a software-based computer
- It runs on top of a hypervisor
- Multiple VMs can run on a single physical server
- VMs are the foundation of cloud services like GCP Compute Engine
